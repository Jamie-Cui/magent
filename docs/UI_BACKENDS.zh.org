Magent 现在把 UI 明确当作 backend boundary。默认 backend 是 ~agent-shell~ ，通过进程内 ACP adapter 连接。旧 workspace/compose UI 保留为一个 legacy backend 文件，方便未来移除时不破坏 agent-shell + ACP + runtime 主路径。

* 模块

- ~magent-ui.el~ 是公共命令 router、logger、compatibility shim 和 ~magent-ui-backend~ 切换点。
- ~magent-agent-shell.el~ 注册 Magent ~agent-shell~ 配置，并创建进程内 ACP client。
- ~magent-acp.el~ 实现进程内 ACP request、notification、response、permission 和 session/update adapter。
- ~magent-runtime-api.el~ 是 UI-neutral runtime/session API，供 UI backends 调用。
- ~magent-runtime-queue.el~ 负责全局单执行队列和 session-scoped cancellation。
- ~magent-ui-legacy.el~ 包含旧 special-mode workspace、compose buffer、timeline rendering、transient menu 和 legacy prompt dispatch。

* 默认流程

1. ~magent-dwim~ 和普通 ~magent-ui-dispatch-prompt~ 等公共命令进入 ~magent-ui.el~ 。
2. 当 ~magent-ui-backend~ 为 ~agent-shell~ ，并且没有 legacy-only per-request agent override 时，命令路由到 ~magent-agent-shell.el~ 。
3. ~agent-shell~ 调用 ~magent-acp.el~ 中的 Magent 进程内 ACP client。
4. ACP session/prompt 提交到 ~magent-runtime-api.el~ 。
5. runtime queue 每次只启动一个 active turn，创建带 ~:ui-visibility 'none~ 的 ~magent-request-context~ ，并调用 ~magent-agent-run-turn~ 。
6. agent loop 发出 Magent-native observer events； ~magent-acp.el~ 将其转换成 ACP ~session/update~ 消息给 agent-shell。
7. ACP prompt request 会保持 pending，直到对应 Magent turn completed、failed 或 cancelled。

* Legacy 边界

~magent-ui-legacy.el~ 按需懒加载。普通 agent-shell 使用不应该加载它。legacy-only 命令仍然按需加载它，包括旧 workspace/compose buffer、legacy transient menu variants，以及依赖 per-request agent override 的命令。

核心 runtime 路径不能依赖 ~magent-ui-legacy.el~ 。新的 backend-facing 行为应放在 ~magent-runtime-api.el~ 或 ~magent-acp.el~ ，agent-shell-specific 行为应放在 ~magent-agent-shell.el~ 。

* 当前限制

- 第一版对 ~agent-shell~ 和 ~acp~ 是硬依赖。
- runtime execution 是全局单执行队列。
- cancellation 是 session-scoped：一个 ACP session 的 cancellation 会取消该 session 的 active 和 queued submissions，但不会移除其他 session 的 queued work。
- agent-shell backend 支持 per-request instruction skill overrides 和 skill commands。per-request agent overrides 仍然走 legacy dispatch path，直到 agent-shell 暴露等价控件。
- ~magent-ui-backend~ 可以设为 ~legacy~ ，但默认值是 ~agent-shell~ 。
